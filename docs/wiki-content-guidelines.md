# 트루핑키스 위키 지식 추가 지침

트루핑키스 위키는 학생이 교과 지식과 생활 지식을 안전하게 찾도록 돕는 교육용 위키입니다. 새 지식은 많아지는 것보다 정확하고 저작권 문제가 없게 쌓이는 것이 우선입니다.

## 0. 용어 기준

- 사용자가 읽고 찾는 개별 항목은 `지식`으로 부릅니다. 예: `지식 목록`, `오늘의 지식`, `함께 보면 좋은 지식`
- 출처로 확인한 원천은 `자료`로 부릅니다. 예: `확인한 자료`, `이 자료로 확인한 내용`
- 앱 내부 코드 변수는 `doc`처럼 기존 이름을 유지할 수 있습니다. 지식 원본 파일은 `data/source/knowledge` 아래에서 관리하고, 화면용 JSON은 `data/generated`에 자동 생성합니다.
- `개념 지식`, `용어 지식`처럼 경계가 애매한 종류 구분은 사용하지 않습니다. 지식은 교과목, 분류, 태그, 검색 보조어, 함께 보면 좋은 지식으로 찾게 합니다.
- 외부 출처의 이름에 `문서`, `자료`, `보고서`가 들어간 경우에는 원래 제목을 바꾸지 않습니다.
- 새 화면 문구를 만들 때는 같은 기능을 `문서`, `지식`, `항목`으로 섞어 부르지 않습니다.
- 모든 지식의 첫머리에는 `definition`을 사용해 그 지식을 대표하는 `핵심 뜻`을 보여 줍니다. `definition`은 학생이 처음 읽고 바로 이해할 수 있을 만큼 간결해야 합니다.
- `summary`는 첫 화면, 목록, 검색 결과 카드에서 쓰는 `짧은 소개`입니다. 상세 화면의 본문 상단에서는 보여 주지 않으므로, `definition`과 같은 문장을 반복하지 말고 그 지식이 왜 중요한지 한 문장으로 알려 줍니다.
- 한자·영어 표기는 `termNotations`에 따로 둡니다. 제목이나 핵심 뜻보다 작게 보이는 보조 정보이며, 동음이의어 구분이나 교과 용어 검색에 필요한 경우부터 적용합니다.
- `termNotations`는 `term`, `hanja`, `english`를 사용합니다. 예: 소화(消化, digestion), 소화(消火, fire extinguishing), 분자(分子, molecule), 분수의 분자(分子, numerator).
- 한자나 영어가 초등학생에게 어려운 용어라도, 뜻 구분에 도움이 되는 경우에는 보조 표기로 둡니다. 단, 본문 설명은 한국어가 중심이어야 합니다.

## 1. 출처 원칙

- 공공기관, 정부기관, 공공 교육기관, 박물관·과학관·국가기관 자료를 우선 사용합니다.
- 새 지식은 이용 조건상 참고·인용·재작성에 문제가 없는 자료에서만 확인합니다.
- 가져오거나 참고해도 되는지 확실하지 않은 자료는 사용하지 않습니다.
- 사이트에는 교육용 서비스이며 저작권자의 삭제 요청이 있을 경우 관련 게시물이 삭제될 수 있다는 안내를 표시합니다.
- 출처는 본문 사실을 확인하기 위한 자료입니다. 원문 문장, 이미지, 표, 그래프, 활동지, 도표를 그대로 가져오지 않습니다.
- 모든 문장에 외부 출처가 필요한 것은 아닙니다. 일반적인 배경 설명, 쉬운 비유, 예시, 퀴즈 문항은 AI와 작성자의 일반 지식으로 새로 쓸 수 있습니다.
- AI와 작성자의 일반 지식으로 새로 쓴 문장은 출처 항목으로 만들지 않습니다. 출처 항목에는 실제로 확인한 자료만 적습니다.
- 연도, 법령, 건강·안전, 금융, 역사적 사건, 특정 기관·인물처럼 틀리면 문제가 큰 사실은 공식 자료로 확인한 뒤 씁니다.
- AI는 작성 도구일 수 있지만 출처가 아닙니다. AI가 문장을 만들었다는 이유만으로 출처 항목을 만들지 않습니다.
- AI 도움으로 초안을 작성하거나 퀴즈·예시를 정리한 지식은 사이트 화면에서 AI 작성 도움 안내를 명확히 보여 줍니다.
- `sources`에는 실제로 사실 확인에 사용한 공식 자료, 공공 자료, 권위 자료만 적습니다.
- `usedFor`에는 자료 제목을 반복하지 말고 “이 자료로 무엇을 확인했는지”를 씁니다.
- 공공누리, 공공데이터, 정부·공공기관 자료라도 이용 조건을 확인하고 출처를 남깁니다.
- `All rights reserved`처럼 저작권 보호 표시가 있거나 재사용 조건이 분명하지 않은 공식 자료는 본문 사실 확인용으로만 참고합니다. 이 경우 원문 문장, 문단 구성, 표, 이미지, 그래프를 가져오지 않고 학생용 새 문장과 새 구조로 작성합니다.
- 상업 백과, 언론 기사, 민간 웹사이트는 보조 확인용으로만 제한적으로 사용합니다. 이용 조건이 불명확하면 새 지식의 핵심 출처로 쓰지 않습니다.
- 나무위키, 개인 블로그, 카페, 커뮤니티, SNS, 광고성 페이지, 출처 없는 요약문은 새 지식의 출처로 쓰지 않습니다.

## 2. 작성 방식

- AI 도움을 받을 수 있지만, 출처로 확인한 사실을 학생용 문장으로 새로 씁니다.
- 새 지식은 하나하나 주제의 범위, 목차, 예시, 퀴즈, 출처를 따로 판단해 작성합니다. 제목만 바꾸고 같은 본문 구조를 반복하는 방식으로 일괄 생성하지 않습니다.
- 기존 지식을 개편할 때도 `data/source/knowledge-order.json` 순서를 기준으로 하나씩 검토합니다. 사용자가 특정 지식을 지정하지 않았다면 앞에서부터 차례대로 진행하고, 문제가 많아 보이는 지식만 골라 고치지 않습니다.
- 한 지식을 검토할 때는 `title`, `definition`, `summary`, 목차, 본문 첫 문장, 예시, 비교 설명, 퀴즈, 관련 지식, 출처를 한 번에 확인한 뒤 그 지식의 수정 여부를 결정합니다.
- 여러 지식을 한꺼번에 검색해 문제 후보를 뽑는 것은 보조 조사로만 허용합니다. 실제 수정 여부는 반드시 각 지식 파일을 처음부터 끝까지 읽고 결정합니다.
- `관계없다`, `이름만 알면 된다`, `중요합니다`, `연결해 볼 수 있습니다` 같은 표현을 일괄 검색해도, 검색 결과를 한꺼번에 치환하지 않습니다. 해당 지식의 문맥에서 필요한 문장인지 먼저 판단합니다.
- 지식 본문, 퀴즈, 예시를 수정할 때는 한 파일의 수정과 확인을 끝낸 뒤 다음 파일로 넘어갑니다. 여러 파일을 열어 놓고 같은 문장틀을 반복 적용하지 않습니다.
- 자동 스크립트는 JSON 형식 점검, 지식 목록 확인, 생성 파일 빌드처럼 기계적인 작업에만 씁니다. `data/source/knowledge`의 본문·목차·퀴즈 문장을 자동으로 대량 재작성하는 데 쓰지 않습니다.
- 검토 결과 현재 구조가 자연스러우면 억지로 문장을 바꾸지 않습니다. 대신 다음 지식으로 넘어가기 전에 왜 그대로 두어도 되는지 확인합니다.
- `definition`과 본문 첫 문장이 가까운 위치에서 같은 뜻을 반복하면 본문 첫 문장을 조정합니다. 다만 본문 흐름상 뜻을 다시 짚어야 자연스러운 경우에는 더 구체적인 기준, 예시, 범위를 붙여 반복처럼 보이지 않게 씁니다.
- `summary`는 상세 화면 설명문이 아니라 목록용 소개입니다. 본문 목차와 `definition`만으로 충분히 안내되는 내용을 `summary`에 다시 쓰지 않습니다.
- 새 지식을 만들기 전에는 기존 지식과 겹치는지 확인합니다. 독립된 지식으로 두는 편이 나은지, 기존 지식 안에서 보강하는 편이 나은지, 여러 개념을 한 지식으로 묶는 편이 나은지 먼저 정합니다.
- 여러 개념을 묶을 때는 학생이 함께 이해해야 하는 이유가 분명해야 합니다. 단순히 같은 교과나 같은 분야라는 이유만으로 억지로 묶지 않습니다.
- 분리한 지식은 `함께 보면 좋은 지식`으로 연결해 학생이 다음 지식으로 이동할 수 있게 합니다.
- 외부 자료를 참고하지 않고 일반 지식으로 설명을 작성할 때도 원문을 떠올려 바꿔 쓰는 방식이 아니라 처음부터 학생용 새 문장으로 씁니다.
- AI가 만든 설명, 예시, 퀴즈는 사람이 다시 읽고 어색한 표현, 억지 예시, 사실 확인이 필요한 문장을 점검합니다.
- 한 문장씩 바꿔 쓰는 수준의 베끼기를 하지 않습니다. 문단 구조와 예시도 새로 구성합니다.
- 인물, 사건, 국가, 제도는 `훌륭한`, `위대한`, `영웅`, `최고`, `최악` 같은 평가성 표현보다 확인 가능한 행동, 제도, 배경, 결과를 중심으로 씁니다.
- 역사 인물은 칭찬이나 비난을 먼저 붙이지 말고, 어떤 시대 조건에서 어떤 선택과 활동을 했으며 그 결과가 무엇이었는지 설명합니다.
- 지식의 깊이는 어려운 말을 늘리는 방식이 아니라 원인, 조건, 과정, 결과, 한계, 다른 개념과의 차이를 차근차근 보여 주는 방식으로 높입니다.
- 초등 교과와 연결되는 개념은 쉬운 설명, 생활 예시, 헷갈리기 쉬운 점, 퀴즈를 함께 씁니다.
- 교과 지식은 가능한 한 2022 개정 교육과정의 내용 영역과 성취기준 흐름을 참고해 정리합니다. 성취기준 원문이나 표를 복제하지 말고, 해당 지식이 어떤 성취기준 학습에 도움이 되는지를 학생용 새 문장으로 씁니다.
- 중학생이나 성인이 볼 수 있는 개념이라도 말투는 학생이 이해할 수 있게 유지합니다.
- 건강, 안전, 법, 금융, 개인정보처럼 중요한 주제는 “공식 자료와 보호자·선생님 확인이 필요하다”는 맥락을 넣습니다.
- 학생용 지식이라고 해서 모든 예시를 학교, 학급, 숙제, 친구 관계로 연결하지 않습니다.
- 학교폭력, 급식처럼 학교 맥락이 주제 자체인 경우를 제외하면 일상, 사회, 과학, 경제, 공공생활처럼 개념에 맞는 장면을 우선 씁니다.
- 예시는 억지로 친근하게 만들기보다 해당 개념을 정확히 이해하는 데 필요한 상황이어야 합니다.

### 2-1. 기존 지식 개편 기준

기존 지식을 고칠 때의 목표는 “전부 새로 쓰기”가 아니라, 이미 맞는 부분은 보존하고 어린이백과 기준에 어긋나는 부분만 고치는 것입니다.

- 기존 지식은 먼저 `유지 / 부분 수정 / 재구성` 중 하나로 판정합니다.
- `유지` 판정이면 문장을 더 예쁘게 만들기 위해 억지로 바꾸지 않습니다.
- `부분 수정`은 중복 문장, 모호한 문장, 어색한 예시, 퀴즈 보기, 출처 표기처럼 문제가 있는 부분만 고칩니다.
- `재구성`은 제목 범위, 목차 흐름, 본문 구조, 퀴즈가 함께 맞지 않을 때만 선택합니다.
- 기존 `id`는 가능하면 유지합니다. 제목이 바뀌어도 검색·관련 지식 연결이 깨질 수 있으므로 `id` 변경은 마지막 수단입니다.
- 제목이나 범위를 바꾸면 `summary`, `definition`, `aliases`, `keywords`, `searchContexts`, `related`, `categoryPaths`, `curriculum-map` 연결도 함께 확인합니다.
- 기존 출처가 여전히 맞고 이용 조건에 문제가 없으면 유지합니다. 새로 추가하거나 바꾼 사실이 있으면 그 사실에 맞는 출처만 보강합니다.
- 본문을 고치면 해당 지식의 퀴즈도 반드시 다시 읽습니다. 본문에서 사라진 사실을 묻는 문제는 수정하거나 삭제합니다.
- 기존 지식의 일부가 다른 지식으로 분리되는 것이 낫다면, 바로 새 파일을 만들기 전에 분리 이유와 새 지식 범위를 먼저 정합니다.

기존 지식 개편 기록은 아래처럼 남깁니다.

```text
- id:
- 기존 판정: 유지 / 부분 수정 / 재구성
- 보존한 부분:
- 고친 부분:
- 퀴즈 변경 여부:
- 출처 변경 여부:
```

### 2-2. 새 지식 추가 기준

새 지식은 만들기 전에 “정말 별도 지식으로 둘 이유가 있는지”를 먼저 통과해야 합니다. 기존 지식 개편보다 진입 기준이 더 엄격합니다.

새 지식을 만들기 전 아래 질문에 답합니다.

| 질문 | 통과 기준 |
| --- | --- |
| 별도 지식인가? | 기존 지식의 한 단원으로 충분하지 않고, 학생이 독립적으로 검색할 만한 주제이다. |
| 범위가 적당한가? | 한 지식 안에서 뜻, 배경·원리, 예시, 오해, 퀴즈를 다룰 수 있는 크기이다. |
| 겹치는 지식은 없는가? | 기존 지식과 중복되지 않거나, 중복된다면 기존 지식을 보강하는 편보다 새 지식이 낫다. |
| 출처가 가능한가? | 공식 자료, 공공 자료, 권위 자료 또는 일반 지식으로 안전하게 새 문장 작성이 가능한 근거가 있다. |
| 저작권 위험은 없는가? | 원문 문장, 표, 이미지, 그래프를 가져오지 않고 학생용 새 문장으로 쓸 수 있다. |
| 분류가 명확한가? | 관련 교과, 큰 주제, 작은 주제, 태그, 관련 지식이 정해진다. |
| 퀴즈가 자연스러운가? | 본문 핵심을 확인하는 문제를 억지 없이 만들 수 있다. |

새 지식 작성 전에는 아래 설계 메모를 먼저 정합니다.

```text
- 새 지식 제목:
- 별도 지식으로 둘 이유:
- 포함할 범위:
- 제외할 범위:
- 확인할 자료:
- 예상 목차:
- 함께 보면 좋은 지식:
- 퀴즈로 확인할 핵심:
```

새 지식은 설계 메모가 불분명하면 만들지 않습니다. 특히 출처 확인이 필요한 역사 연도, 법령, 건강·안전, 금융, 개인정보, 특정 기관·인물 정보는 자료 확인 전 본문을 쓰지 않습니다.

## 3. 어린이백과 품질 기준

어린이백과의 목표는 학생에게 특정한 생각을 요구하는 것이 아니라, 학생이 사실과 개념을 정확히 이해할 수 있게 돕는 것입니다. 쉬운 말투를 쓰되 백과의 성격을 잃지 않아야 합니다.

실제 어린이백과와 지식백과의 공통점은 아래처럼 정리해 적용합니다.

- 어린이백과는 같은 탐색 구조를 유지합니다. 지식마다 제목, 첫 설명, 목차, 본문, 보조 자료, 관련 지식, 출처가 같은 위치에서 보여야 합니다.
- 본문은 낱말뜻만 말하지 않고 주제의 범위, 작동 방식, 배경, 결과, 관련 개념까지 설명합니다.
- 초급 독자에게 맞추되 내용은 얕게 만들지 않습니다. 어려운 개념은 빠뜨리지 말고 작은 단계로 나누어 설명합니다.
- 실제 지식백과형 항목은 대체로 `정의`, `요약`, `키워드`, `본문 목차`, `관련 항목`, `참고문헌 또는 출처`가 분리됩니다. 트루핑키스 위키도 이 기능을 갖추되, 학생 화면에서는 용어를 쉽게 바꿔 보여 줍니다.
- 나무위키식 장점은 세부 목차와 관련 항목 연결이 풍부하다는 점입니다. 다만 비공식 말투, 밈, 취소선, 평가성 표현, 출처가 약한 서술은 트루핑키스 위키에 쓰지 않습니다.
- 한국민족문화대백과식 장점은 항목 첫머리에 분류, 시대, 핵심 정의, 관련 키워드, 참고 자료가 분명하다는 점입니다. 다만 어린이백과에서는 전문 용어를 그대로 늘어놓기보다 본문에서 쉬운 말로 풀어 씁니다.
- 외부 백과의 목차명과 문장 구조를 그대로 가져오지 않습니다. 참고한 것은 구성 원칙이며, 실제 문장은 트루핑키스 위키의 학생용 새 문장으로 씁니다.
- 단, 모든 비슷함을 억지로 피하지 않습니다. 같은 사실을 설명하면 `언제`, `누가`, `무엇을`, `어떤 결과를` 같은 기본 문장 구조가 자연스럽게 비슷해질 수 있습니다.
- 저작권 위험은 일반적인 사실 배열이 아니라 원문 표현, 독특한 문장, 세부 목차 배열, 예시, 표, 해설 구조를 거의 그대로 옮길 때 커집니다. 사실을 확인한 뒤 백과식으로 새로 쓰되, 어색한 우회 표현을 만들지 않습니다.
- 공식 명칭, 연도, 제도명, 사건 순서, 과학 용어는 정확성이 우선입니다. 원문과 같은 단어를 피하려고 부정확한 대체어를 만들지 않습니다.
- 과학 지식은 `도입 설명 → 원리나 과정 → 조건과 예외 → 관련 현상`으로 깊어져야 합니다.
- 역사·사회 지식은 `뜻이나 사건 개요 → 시대·제도 배경 → 흐름과 결과 → 관련 인물·기관·자료`로 이어져야 합니다.
- 역사 국가·시대 지식은 `성립 → 성장·전성기 → 제도·문화 → 쇠퇴·다음 시대로의 변화`를 개별적으로 확인합니다. 모든 항목에 같은 목차를 복사하지 말고, 해당 국가나 시대에 실제로 필요한 갈래만 둡니다.
- 역사 사건·인물·국가를 보강할 때는 이름만 나열하지 않습니다. 관련 인물, 날짜, 장소, 사건의 결과가 본문에서 실제 역할과 함께 설명되어야 합니다.
- 신라, 고려처럼 긴 시기를 다루는 지식은 앞뒤 시대와의 연결을 사실 중심으로 적습니다. 예를 들어 신라 말은 견훤, 궁예, 왕건, 후백제, 후고구려, 고려 건국, 후삼국 통일을 필요한 범위 안에서 함께 확인합니다.
- 미디어, 지도, 표, 연표, 관련 지식은 장식이 아니라 본문 이해에 필요한 경우에만 둡니다.
- 출처와 인용 정보는 학생이 자료의 근거를 확인할 수 있게 분리해 표시합니다.

### 3-0. 한 지식의 합격 기준

기존 지식을 점검할 때 아래 항목을 모두 통과해야 `어린이백과에 맞는 지식`으로 봅니다.

| 항목 | 합격 기준 | 수정이 필요한 상태 |
| --- | --- | --- |
| 제목 | 지식의 범위를 정확히 가리킨다. 개념, 사건, 인물, 제도, 현상 중 무엇인지 분명하다. | 제목이 사건인지 개념인지 흐리거나, 여러 개념을 억지로 묶었다. |
| 핵심 뜻 | 한두 문장 안에 분류와 핵심 기준이 들어 있다. | 본문 첫 문장과 거의 같거나, 너무 짧아 무엇인지 구분되지 않는다. |
| 목록 소개 | 검색 결과에서 읽을 가치와 범위를 알려 준다. | 핵심 뜻을 그대로 반복하거나 홍보 문구처럼 보인다. |
| 목차 | 지식의 실제 내용이 읽을 순서대로 배치되어 있다. | `보는 법`, `흐름`, `조심할 점`처럼 학습 안내나 메타 설명에 머문 제목이 많다. |
| 본문 | 사실, 기준, 과정, 예시, 차이를 차례대로 설명한다. | 안내 문구, 관점 제시, 같은 말 반복, 두루뭉실한 문장이 많다. |
| 예시 | 개념의 기준을 분명하게 보여 준다. | 학교생활 예시로 억지 연결하거나, 개념 이해에 새 정보를 주지 않는다. |
| 보조 구조 | 비교, 연표, 지도, 표가 본문 이해에 실제로 필요하다. | 모든 지식에 같은 보조 구조를 붙이거나 정확도가 낮은 자료를 넣었다. |
| 퀴즈 | 본문 핵심을 묻고, 오답도 그럴듯한 착각을 반영한다. | `관계없다`, `이름만 알면 된다` 같은 빈약한 오답이 반복된다. |
| 출처 | 확인한 사실과 출처가 대응된다. | 출처가 없거나, AI가 쓴 일반 설명까지 출처처럼 표시했다. |

### 3-1. 백과형 문장 기준

- 문장은 `무엇인가`, `어떻게 작동하는가`, `언제·어디서·누가·무엇을 했는가`, `원인과 결과가 무엇인가`, `다른 개념과 무엇이 다른가`를 설명해야 합니다.
- `이렇게 이해해야 합니다`, `넓은 관점으로 보아야 합니다`, `함께 생각해 볼 수 있습니다`, `중요하게 바라보아야 합니다`처럼 사고의 방향을 강하게 지시하는 문장은 줄입니다.
- 다만 사실을 오해하기 쉬운 주제에서는 판단 기준을 분명히 씁니다. 예: `하루 날씨와 기후는 구분해야 합니다`는 가능하지만, 같은 뜻을 여러 번 반복하지 않습니다.
- `좋은`, `나쁜`, `훌륭한`, `위대한`, `아쉬운`, `문제적인` 같은 평가어는 사실 설명에 꼭 필요하지 않으면 쓰지 않습니다.
- 평가가 필요한 경우에도 “왜 그렇게 평가되는지”를 사실 근거로 설명합니다. 예: `국민의 기본권을 넓히는 방향으로 제도가 바뀌었습니다`처럼 확인 가능한 변화로 씁니다.
- 백과 문장은 짧고 단정해야 하지만, 지나치게 단순화해 틀린 말이 되면 안 됩니다. 예외, 조건, 범위가 중요한 개념은 함께 적습니다.
- `~와 연결해 볼 수 있습니다`, `~도 생각해 볼 수 있습니다`, `~을 넓게 바라보아야 합니다` 같은 문장은 가능하면 사실 문장으로 바꿉니다. 예: `기후 변화는 농업, 건강, 물 사용, 전기 사용에 영향을 줍니다.`
- `중요합니다`만으로 끝나는 문장은 이유를 붙입니다. 예: `출처 확인이 중요합니다`보다 `출처를 확인하면 누가 언제 어떤 자료를 바탕으로 말했는지 알 수 있습니다`처럼 씁니다.
- `~일 수 있습니다`가 반복되면 모호한 문장으로 봅니다. 가능성이 아니라 확인된 범위, 조건, 예시를 씁니다.
- 어린이백과의 쉬운 설명은 `학생들이 이해하기 위해서는`, `알아 두면 좋습니다`, `이해하는 데 도움이 됩니다`, `~와 연결해 생각할 수 있습니다` 같은 교육 안내형 문장으로 길게 풀어 쓰지 않습니다. 필요한 사실을 직접 씁니다.
- `태풍은 강한 바람과 많은 비를 동반합니다`처럼 사실을 바로 쓰고, `학생들이 태풍의 위험을 이해하기 위해서는 바람과 비가 함께 나타날 수 있다는 점을 알아두는 것이 중요합니다`처럼 독자의 학습 방향을 설명하는 문장은 쓰지 않습니다.
- “중요하다”와 “필요하다”는 안전, 건강, 법, 실험 절차처럼 실제 조건이나 위험을 설명할 때만 씁니다. 개념의 가치나 학습 필요성을 말하려고 붙인 문장은 사실 설명으로 바꿉니다.
- `흐름에 포함됩니다`, `이어졌습니다`, `관련됩니다`, `영향을 보여 줍니다`, `기준이 됩니다`처럼 관계만 말하고 끝나는 문장도 피합니다. 관계를 써야 할 때는 무엇이 무엇에 어떤 방식으로 작용했는지 구체적으로 씁니다.
- 예: `부여는 고대 국가 흐름에 포함됩니다`가 아니라 `부여는 만주 지역에서 성장했고, 고구려와 백제의 건국 이야기에도 부여 계통 인물이 등장합니다`처럼 씁니다.
- 예: `동학 농민 운동은 근대사의 변화와 이어졌습니다`가 아니라 `동학 농민 운동 뒤 조선 정부는 갑오개혁을 추진했고, 일본군의 개입과 탄압은 항일 의병 활동이 커지는 배경 중 하나가 되었습니다`처럼 씁니다.

### 3-1-1. 목차 기준

- 목차는 본문을 읽는 방법이 아니라 본문에서 다루는 실제 내용을 보여 주어야 합니다.
- `자료 보는 법`, `정보 읽기`, `조심할 점`, `중요한 흐름`, `고대사 속 자리`, `왕 이름보다 중요한 흐름`, `관련해서 생각하기`처럼 어느 지식에나 붙일 수 있는 메타 설명식 목차는 쓰지 않습니다.
- `예보를 보고 준비하는 법`, `정보를 확인하는 방법`, `이해할 때 중요한 점`, `생활과 연결하기`처럼 독자에게 무엇을 하라고 안내하는 목차는 먼저 백과형 지식 목차로 바꿉니다. 예: `태풍 정보 보는 법`보다 `태풍 예보와 특보`, `기후 변화 대응 방법`보다 `온실가스 감축과 기후 적응`이 낫습니다.
- 목차 제목은 `대상`, `과정`, `조건`, `원인과 결과`, `비교 대상`, `시대·지역·인물·제도`가 드러나게 씁니다.
- 목차 제목만 보고도 그 단원에서 어떤 지식을 얻는지 알 수 있어야 합니다. 예: `태풍 정보 보는 법`보다 `태풍 예보의 정보`, `통계 자료 보는 법`보다 `조사 대상과 평균`, `고대사 속 부여의 자리`보다 `부여와 주변 고대 국가`가 낫습니다.
- 목차가 `뜻`, `특징`, `영향`, `관련 인물`, `퀴즈`처럼 너무 넓은 말만 반복되면 해당 지식의 고유한 내용이 드러나도록 바꿉니다. 예: `영향`보다 `강화도 조약 뒤의 개항`, `관련 인물`보다 `대조영과 발해 건국`처럼 씁니다.
- 큰 목차는 너무 세밀한 항목을 나열하지 않고, 작은 목차는 큰 목차의 내용을 실제로 나누어야 합니다. 큰 목차와 작은 목차가 같은 말을 반복하면 합치거나 제목을 바꿉니다.
- 목차를 클릭했을 때 이동하는 본문에는 목차 제목에 해당하는 설명이 충분히 있어야 합니다. 본문이 1문장뿐이거나 제목과 같은 말을 되풀이하면 목차로 세울 만큼 내용이 있는지 다시 봅니다.
- 역사 지식의 목차는 가능하면 시간 순서, 원인과 결과, 관련 인물·제도·사건의 관계가 보이게 구성합니다. 단, `역사의 흐름`, `이어지는 흐름`처럼 관계만 말하는 제목은 쓰지 않고 `정묘호란과 병자호란`, `개항과 대한제국`, `고구려 유민과 발해`처럼 실제 내용을 씁니다.
- 과학 지식의 목차는 현상, 원리, 조건, 관찰 결과, 비슷한 개념의 차이가 보이게 구성합니다. `관찰할 때 조심할 점`보다 `그림자 관찰 조건`, `모형을 볼 때 조심할 점`보다 `입자 모형의 역할과 한계`처럼 씁니다.
- 사회·경제 지식의 목차는 제도의 뜻, 작동 방식, 자료 기준, 생활·산업과의 관계, 비슷한 용어와의 차이가 보이게 구성합니다. `환율 자료 보는 법`보다 `환율 변동과 경제 정보`, `물가 뉴스 보는 법`보다 `물가 통계와 뉴스`처럼 씁니다.

### 3-1-2. 메타인지형 설명 금지 기준

메타인지형 설명은 지식 자체를 설명하지 않고, 독자에게 `어떻게 보아야 하는지`, `무엇을 확인해야 하는지`, `왜 중요하게 생각해야 하는지`를 길게 안내하는 문장입니다. 어린이백과에는 필요한 사실을 직접 씁니다.

- `~을 확인해야 합니다`, `~을 살펴야 합니다`, `~을 따르는 것이 중요합니다`가 반복되면 안내문으로 보입니다. 안전·건강·법처럼 행동 정보가 필요한 경우에도 먼저 현상과 위험의 사실 관계를 쓰고, 행동 정보는 별도 단원에서 짧게 둡니다.
- `중심 경로뿐 아니라 강풍 반경도 확인해야 합니다`보다 `태풍 예보에는 예상 진로, 중심기압, 최대풍속, 강풍 반경이 포함됩니다`처럼 정보의 구성 자체를 설명합니다.
- `기후 변화는 긴 흐름 속에서 보아야 합니다`보다 `기후 변화는 여러 해의 기온, 강수량, 바다 높이, 얼음 변화 같은 자료로 판단합니다`처럼 판단에 쓰이는 자료를 씁니다.
- `공공기관 안내를 따르는 것이 중요합니다`보다 `태풍특보, 호우특보, 강풍특보, 재난문자는 지역별 위험 정보를 전달합니다`처럼 제도와 정보의 역할을 설명합니다.
- 한 문단이 `중요하다`, `필요하다`, `도움이 된다`, `확인해야 한다`로 끝나면 그 문장이 실제 지식인지 행동 안내인지 다시 판정합니다.
- `중요합니다`, `조심해야 합니다`, `연결할 수 있습니다`, `도움이 됩니다`, `알아 두면 좋습니다`는 백과 본문을 얕게 만드는 대표 신호입니다. 해당 문장에는 보통 사실 정보가 빠져 있으므로 날짜, 기관, 인물, 조건, 원인, 결과, 기준, 예시 중 무엇을 설명해야 하는지 먼저 확인합니다.
- `중요합니다`를 써야 할 만큼 핵심이라면 왜 핵심인지 사실로 바꿉니다. 예: `기상특보 확인이 중요합니다`보다 `기상특보는 호우, 강풍, 태풍처럼 지역별 위험 정보를 알립니다`처럼 씁니다.
- `조심해야 합니다`는 안전 행동을 직접 안내해야 하는 지식에서만 짧게 씁니다. 과학·역사·사회 백과 본문에서는 위험 조건이나 제도 정보를 먼저 씁니다.
- `연결할 수 있습니다`, `관련됩니다`, `영향을 줍니다`만으로 끝나는 문장은 관계가 비어 있는 문장입니다. 무엇이 무엇에 어떤 방식으로 작용했는지 구체화합니다.
- `깊이 있게 이해할 수 있습니다`, `생각해 볼 수 있습니다`, `함께 살펴볼 수 있습니다`처럼 학습 행위를 설명하는 문장은 삭제하거나 사실 문장으로 바꿉니다.
- 백과 본문에는 독자가 무엇을 해야 하는지가 아니라 대상 자체의 정보가 먼저 나와야 합니다. 학생용 문장이어도 교사의 설명문이나 학습 안내문처럼 쓰지 않습니다.

### 3-1-3. 사실 밀도 기준

지식의 깊이는 문장 수가 아니라 사실 밀도로 판단합니다. 한 지식이 너무 가볍게 느껴지면 아래 정보를 개별 항목에 맞게 보강합니다.

- 역사 사건: 연도, 장소, 배경 세력, 직접 원인, 전개 과정, 결과, 관련 인물·단체, 이후 변화.
- 역사 인물: 활동 시기, 활동 지역, 소속·직책, 주요 사건, 남긴 저술·제도·작품·자료, 활동의 결과.
- 국가·시대: 성립 시기, 중심 지역, 정치 제도, 대외 관계, 사회·문화 특징, 변화와 멸망·계승.
- 과학 현상: 관찰되는 현상, 원인, 조건, 과정, 측정 기준, 비슷한 개념과의 차이, 대표 예시.
- 사회 제도: 담당 기관, 절차, 법·규칙의 범위, 시민 생활에서 나타나는 모습, 권리와 책임.
- 경제·금융: 돈이 움직이는 과정, 위험과 수익의 조건, 비슷한 용어와 차이, 실제 통계나 시장에서 쓰이는 기준.
- 디지털·미디어: 작동 방식, 입력과 출력, 사용되는 데이터, 한계, 오류가 생기는 조건, 확인 기준.
- 한 단원이 세 문장 이하라도 괜찮지만, 그 세 문장이 모두 `뜻`, `중요성`, `연결`만 말하면 깊이가 부족합니다.
- 날짜와 인물은 무조건 많이 넣지 않습니다. 그 지식을 구분하는 데 필요한 날짜·인물·사건·기관만 넣습니다.
- 새 사실을 보강할 때는 기존 출처로 확인 가능한지 먼저 보고, 확인이 부족하면 공식·권위 자료를 추가합니다. 출처 없이 특정 연도·사건·인물을 새로 넣지 않습니다.

### 3-2. 어린이백과다운 깊이

- 쉬운 설명은 정보량을 줄이는 것이 아니라, 어려운 개념을 순서대로 나누는 것입니다.
- 한 지식은 `핵심 뜻`, `중요한 기준`, `과정이나 배경`, `대표 예시`, `헷갈리기 쉬운 점` 중 그 지식에 필요한 요소를 갖추어야 합니다. 모든 요소를 억지로 넣지는 않습니다.
- 본문은 문단 수나 단원 수로 판단하지 않습니다. 짧아도 개념의 범위와 기준이 충분하면 유지하고, 길어도 같은 말을 반복하거나 사실이 부족하면 수정합니다.
- 각 작은 단원에는 그 단원을 세운 이유가 분명해야 합니다. 연도, 장소, 기관, 원인, 과정, 조건, 대표 사례, 비교 대상, 자료 기준, 결과 중 해당 주제에 필요한 사실만 씁니다.
- 한 지식 안에는 제목과 정의를 풀어 쓴 문장만 반복하지 말고, 학생이 새로 알 수 있는 백과식 설명이 들어가야 합니다.
- 사실 데이터가 적은 지식은 단원을 늘리거나 문장을 길게 만드는 방식보다, 빠진 핵심 기준과 필요한 사실을 정확히 보강합니다. 설명할 갈래가 적은 지식은 짧게 유지할 수 있습니다.
- 과학 지식은 현상, 원리, 관찰·실험 조건, 비슷한 개념과의 차이를 중심으로 정리합니다.
- 사회 지식은 제도의 뜻, 작동 방식, 관련 기관, 시민 생활과의 연결을 사실 중심으로 정리합니다.
- 역사 지식은 시대 배경, 사건의 흐름, 관련 인물·제도·결과를 시간 순서와 원인·결과 중심으로 정리합니다.
- 경제 지식은 용어의 뜻, 돈이나 시장에서 일어나는 과정, 비슷한 금융·경제 용어와의 차이를 함께 설명합니다.
- 인물 지식은 출생·활동·업적만 나열하지 말고, 활동한 시대와 그 활동이 남긴 확인 가능한 결과를 함께 설명합니다.
- 한 지식의 본문이 `뜻 설명`만 하고 끝나면 깊이가 부족한 상태입니다.
- 한 지식의 본문이 `뜻 → 같은 뜻 반복 → 퀴즈`로 이어지면 구조를 다시 짭니다.
- 한 지식이 다른 지식과 비교해야 이해되는 경우에는 비교 단원을 넣습니다. 예: `속력과 속도`, `정지마찰력과 운동마찰력`, `예금·채권·주식·펀드`.
- 한 지식이 시간 흐름으로 이해되는 경우에는 연표를 넣습니다. 예: 역사 사건, 제도 변화, 기술 발전, 산업 발달.
- `역사` 주제에 속하더라도 시간 순서가 중심이 아닌 지식에는 연표를 붙이지 않습니다. 예: 궁궐처럼 공간의 기능·구조·비교가 중심인 지식은 본문 안에서 대표 사례의 시기를 설명하고, 별도 연표는 생략합니다.
- 연표는 단순히 `시작-끝`만 적지 않고, 개념 이해에 필요한 중간 변화와 결과를 포함합니다. 역사 사건·국가·제도·인물 지식은 가능하면 `배경 또는 성립 → 전개 또는 주요 활동 → 변화 또는 결과 → 이후 영향`이 드러나야 합니다.
- 연표 날짜는 `이후`, `오늘날`, `삼국 시대`처럼 흐린 표현만으로 끝내지 않습니다. 정확한 연도를 알기 어려운 고대사·전승은 `기원전 1세기 무렵`, `전승`, `7세기 전반`처럼 불확실성의 성격을 함께 적습니다.
- 연표 설명은 제목을 다시 풀어 쓰는 데 그치지 않습니다. 누가, 어디서, 어떤 제도·사건·자료와 관련되는지 한 문장 안에 구체 정보가 들어가야 합니다.
- 연표 항목 수는 고정하지 않습니다. 다만 시간 흐름이 핵심인 역사 사건·제도·인물 지식에서 2개뿐이면 대체로 부족한 상태로 보고, 빠진 중간 흐름이 있는지 먼저 점검합니다.

### 3-2-1. 분야별 기본 목차 틀

아래 틀은 복사해서 그대로 쓰는 양식이 아니라 점검 기준입니다. 지식마다 필요한 항목만 선택합니다.

| 분야 | 권장 흐름 | 반드시 확인할 것 |
| --- | --- | --- |
| 과학 현상·원리 | 뜻과 관찰 장면 → 원리와 과정 → 조건·예외 → 관련 현상이나 실험 | 원인과 결과가 뒤섞이지 않았는가 |
| 생물·몸 | 구조나 기능 → 작동 과정 → 생활 속 변화 → 헷갈리는 개념 | 의학 조언처럼 단정하지 않았는가 |
| 역사 사건 | 사건 개요 → 배경 → 전개 → 결과와 영향 → 관련 인물·자료 | 평가보다 사실 흐름이 먼저인가 |
| 역사 인물 | 시대 배경 → 주요 활동 → 관련 사건·제도 → 남은 자료나 평가의 범위 | 영웅화하거나 비난하지 않았는가 |
| 국가·시대 | 성립 → 정치·사회·문화 특징 → 대외 관계 → 변화와 멸망·계승 | 전기·후기 흐름이 빠지지 않았는가 |
| 사회 제도 | 뜻 → 담당 기관·절차 → 생활에서 나타나는 모습 → 권리·책임·한계 | 제도 홍보처럼 쓰지 않았는가 |
| 경제·금융 | 뜻 → 돈이 움직이는 방식 → 비슷한 용어와 차이 → 위험·조건 | 투자 조언처럼 보이지 않는가 |
| 디지털·미디어 | 뜻 → 작동 방식 → 사용 예 → 한계·확인 기준 | 도구 사용을 권장하는 말투가 아닌가 |
| 환경 | 뜻 → 원인과 과정 → 영향 → 대응 방법과 한계 | 막연한 캠페인 문구가 아닌가 |

### 3-3. 목차와 구성 기준

- 목차명은 `3-1-1. 목차 기준`을 먼저 통과해야 합니다. 메타 설명식 제목이 있으면 본문을 고치기 전에 목차부터 다시 잡습니다.
- 목차는 지식마다 새로 판단합니다. 모든 지식에 같은 목차 틀을 붙이지 않습니다.
- 첫 단원은 그 지식의 뜻을 다시 쓰는 데 그치지 말고, 학생이 본문을 읽기 전에 알아야 할 핵심 기준을 잡아야 합니다.
- 이미 `핵심 뜻`에서 충분히 설명한 말을 첫 단원 첫 문장에서 그대로 반복하지 않습니다. 필요하면 더 구체적인 범위, 조건, 예시를 붙여 새 정보가 되게 합니다.
- 목차 제목은 `쉽게 풀어보기`, `생활 속 연결`, `더 생각해 보기`처럼 어느 지식에나 붙을 수 있는 말보다 해당 지식의 내용이 드러나는 말로 씁니다.
- 한 단원 안에서 같은 문장을 다른 표현으로 반복하지 않습니다. 반복이 필요하면 앞 문장을 삭제하거나, 뒤 문장에 원인·결과·예외·예시를 추가합니다.
- 비교가 이해에 도움이 되는 지식에는 비교 단원을 둘 수 있습니다. 단, 비교 대상이 실제로 헷갈리는 개념이어야 합니다.
- 연표는 시간 흐름이 개념 이해에 도움이 되는 경우에만 넣습니다. 시간 순서가 핵심이 아닌 과학 원리나 수학 개념에 억지로 넣지 않습니다.
- 큰 단원과 작은 단원의 수는 지식의 범위에 맞게 정합니다. 설명할 갈래가 적은 지식에 단원을 억지로 늘리지 않습니다.
- 작은 단원이 하나뿐이어도 그 단원이 독립된 역할을 하면 유지할 수 있습니다. 반대로 단원이 많아도 역할이 겹치면 합칩니다.
- 본문 길이는 고정하지 않습니다. 한 문장이라도 백과식 설명으로 충분한 경우가 있고, 여러 문장이어도 반복뿐이면 줄입니다.
- 단원 수나 문장 수를 늘려 양을 맞추는 방식은 피합니다. 목차 수가 적어도 설명이 충분하면 유지하고, 목차가 많아도 각 단원이 얕으면 합치거나 보강합니다.
- `뜻`, `개요`, `먼저 알아두기`가 서로 비슷한 역할을 하면 하나로 합칩니다.
- `확인 포인트`, `주의할 점`, `오해하기 쉬운 점`은 실제로 헷갈리는 기준이 있을 때만 둡니다.

### 3-4. 예시 기준

- 예시는 학생에게 친근한 장면보다 개념을 정확히 보여 주는 장면을 우선합니다.
- 학교생활 예시는 주제와 직접 관련이 있을 때만 씁니다.
- 경제, 법, 역사, 과학 지식에는 시장, 가정, 지역사회, 자연 현상, 역사 자료, 공공기관 같은 실제 개념에 가까운 예시를 씁니다.
- 예시는 본문을 부드럽게 보이게 하려고 넣는 장식이 아닙니다. 예시를 읽고 개념의 기준이나 차이가 더 분명해져야 합니다.
- 억지 예시나 너무 좁은 예시는 삭제하고, 대표성이 높은 예시로 바꿉니다.
- 예시가 본문보다 길어져 개념 설명을 가리면 줄입니다.
- 예시는 사실 관계가 확인 가능한 범위에서 씁니다. 실제 사건처럼 보이는 가상 예시는 쓰지 않습니다.

### 3-5. 퀴즈 품질 기준

- 퀴즈는 본문을 읽고 핵심을 이해했는지 확인하는 문제여야 합니다.
- 문제는 선생님이 낼 법한 자연스러운 문장으로 씁니다.
- 정답은 본문에서 확인할 수 있어야 하며, 본문에 없는 사실을 갑자기 묻지 않습니다.
- 오답은 `관계없다`, `이름만 알면 된다`, `언제나 같다` 같은 빈약한 문장으로 만들지 않습니다.
- 좋은 오답은 원인과 결과를 바꾸거나, 시대·대상·조건을 잘못 연결하거나, 비슷한 개념과 혼동한 문장입니다.
- 한 지식 안에서 여러 문제의 오답이 같은 틀로 반복되면 다시 씁니다.
- 지식 내용이 짧으면 1~2문항만 둘 수 있습니다. 문제 수를 맞추려고 억지 문항을 만들지 않습니다.
- 빈칸 문제는 핵심 용어, 연도, 인물, 원리처럼 빈칸 자체가 의미 있을 때만 둡니다.
- 문제 하나는 하나의 기준만 물어야 합니다. `특징이나 예시`, `뜻과 영향`, `원인과 결과`를 한 문제에 섞지 않습니다.
- 정답 보기는 본문 문장을 그대로 복사하기보다 같은 사실을 자연스러운 문제 보기 문장으로 바꿔 씁니다.
- 오답 보기는 너무 황당하면 안 됩니다. 학생이 실제로 헷갈릴 수 있는 시대, 대상, 원리, 절차, 조건을 잘못 연결합니다.
- 모든 보기는 길이와 형식이 어느 정도 비슷해야 합니다. 정답만 길고 자세하면 문제 품질이 낮습니다.
- 해설은 정답만 설명하지 말고, 헷갈리는 오답이 왜 틀렸는지도 한 문장 정도로 짚습니다.
- 새로 쓰거나 크게 고치는 퀴즈에는 `choiceExplanations`를 넣습니다. 정답 해설은 왜 맞는지 설명하고, 오답 해설은 그 보기에 들어 있는 잘못된 시대, 대상, 원인, 결과, 조건을 직접 짚습니다.
- 오답 해설은 `틀린 보기입니다`, `본문과 다릅니다`처럼 범용 문구로 끝내지 않습니다.

### 3-6. 순차 점검 절차

기존 지식을 점검할 때는 아래 순서를 한 지식마다 끝까지 적용한 뒤 다음 지식으로 넘어갑니다.

순차 점검은 `검토 → 판단 → 수정 → 재확인 → 기록`의 한 묶음으로 진행합니다. 이 묶음이 끝나기 전에는 다음 지식을 수정하지 않습니다.

1. 제목이 지식 범위와 맞는지 확인합니다.
2. `definition`이 너무 길거나 본문과 중복되지 않는지 확인합니다.
3. `summary`가 목록용 소개로 기능하는지 확인합니다.
4. 목차가 그 지식에 맞는 순서인지 확인합니다.
5. 본문 문장에 반복, 과도한 안내, 평가성 표현, 모호한 문장이 있는지 확인합니다.
6. 예시가 억지스럽거나 학교생활로만 몰려 있지 않은지 확인합니다.
7. 비교·연표·지도·관련 인물 같은 보조 구조가 필요하거나 불필요한지 판단합니다.
8. 퀴즈의 질문, 정답, 오답, 해설이 본문과 정확히 맞는지 확인합니다.
9. 출처와 저작권 표기가 실제 확인 자료와 맞는지 확인합니다.
10. 고칠 부분이 없으면 “그대로 두는 이유”를 작업 메모에 남기고 다음 지식으로 이동합니다.

각 지식을 점검할 때는 아래 작업 기록을 남깁니다.

```text
- id:
- 판단: 유지 / 부분 수정 / 재구성
- 본문 판단:
- 퀴즈 판단:
- 출처 판단:
- 수정 내용:
- 다음에 볼 지식:
```

점검 결과는 아래 셋 중 하나로 판단합니다.

- `유지`: 제목, 구조, 본문, 퀴즈, 출처가 기준을 통과해 고칠 필요가 거의 없습니다.
- `부분 수정`: 제목이나 큰 구조는 맞지만 반복 문장, 모호한 문장, 퀴즈 보기, 출처 표기 중 일부를 고쳐야 합니다.
- `재구성`: 제목 범위, 목차, 본문 흐름, 퀴즈가 함께 어긋나 있어 지식 전체를 다시 짜야 합니다.

아래 방식은 금지합니다.

- 지식 파일 여러 개를 열어 놓고 같은 목차명이나 문장틀을 동시에 적용하는 방식
- 검색 결과만 보고 본문 전체를 읽지 않은 채 문장을 바꾸는 방식
- 퀴즈 오답을 여러 지식에 같은 패턴으로 복사하는 방식
- “문제가 있을 가능성이 높은 지식”만 골라 보고 나머지는 점검한 것으로 처리하는 방식
- 한 지식의 본문은 고쳤지만 퀴즈와 출처를 확인하지 않고 다음 지식으로 넘어가는 방식

## 4. 역사·인물·사건 추가 기준

- 개념 이해에 도움이 되는 경우에만 추가합니다.
- “처음 만든 사람”, “대표 인물”, “중요한 사건”은 한 사람의 업적으로 단정하지 않습니다.
- 논쟁이 있거나 정치적 해석이 갈리는 내용은 중립적으로 쓰고, 학생용 지식에 꼭 필요하지 않으면 제외합니다.
- 특정 개인이나 집단에게 피해가 될 수 있는 의혹, 평가, 비방성 표현은 넣지 않습니다.

## 5. 지식 구조

분류는 아래 역할을 나누어 씁니다.

- `subjects`: 교과 연결입니다. 화면에는 `관련 교과: 사회, 국어, 도덕`처럼 표시합니다.
- `categoryPaths`: 지식이 속한 분류 경로입니다. 한 지식이 여러 분류에 속할 수 있습니다. 예: `[["역사", "한국사", "조선"], ["과학", "탐구와 측정"]]`
- `mainTopic`: 대표 분류의 첫 칸입니다. 기존 화면과 검색 호환을 위해 유지합니다.
- `subTopicPath`: 대표 분류에서 `mainTopic` 뒤에 이어지는 경로입니다. 기존 화면과 검색 호환을 위해 유지합니다.
- `topicTags`: 여러 지식을 가로질러 묶는 태그입니다. 예: `역사 인물`, `훈민정음`, `문화`, `정치`
- `data/source/curriculum-map.json`: 각 지식이 2022 개정 교육과정의 어떤 교과 내용 영역과 학년군에 연결되는지 따로 관리합니다. 본문 출처와 섞지 않고, 학교 공부에 도움이 되는 방향을 짧게 설명합니다.

예시:

```json
{
  "subjects": ["사회", "국어", "도덕"],
  "categoryPaths": [
    ["역사", "한국사", "조선"],
    ["문화와 표현", "한글과 언어"]
  ],
  "mainTopic": "역사",
  "subTopicPath": ["한국사", "조선"],
  "topicTags": ["역사 인물", "훈민정음", "문화", "정치"]
}
```

카테고리는 지식의 위치를 정하고, 태그는 여러 지식을 연결합니다. `역사`, `한국사`, `조선`처럼 이미 분류 경로로 표시한 말은 태그에 반복해서 많이 넣지 않습니다.
경계에 걸친 지식은 `categoryPaths`에 여러 경로를 넣습니다. 예를 들어 장영실은 `역사 > 한국사 > 조선`과 `과학 > 탐구와 측정`에 동시에 속할 수 있습니다.

본문은 `chapters`에 큰 단원과 작은 단원으로 작성합니다. 권장 흐름은 아래와 같습니다.

1. 지식의 뜻과 핵심 기준
2. 생활 속 장면, 역사적 배경, 우리나라와의 연결, 실험·관찰처럼 지식에 맞는 전개
3. 오해하기 쉬운 부분이나 실제로 확인해야 할 기준
4. `퀴즈`

정보상자는 사용하지 않습니다. 지식 하단에는 `퀴즈`, `함께 보면 좋은 지식`, `확인한 자료`가 자연스럽게 이어져야 합니다.
관련 지식은 본문 목차 안에서 억지로 반복하지 않고, 하단의 `함께 보면 좋은 지식` 영역에서 연결합니다.
재미있는 일화, 옛날 이야기, 기록 속 말은 본문 단원에 억지로 섞지 않고 `storyNotes`에 선택적으로 둡니다. 모든 지식에 억지로 넣지 않으며, 출처로 확인되는 이야기만 사용합니다.
`storyNotes`는 핵심 뜻이나 본문 목차를 대신하지 않는 보조 읽을거리입니다. 상세 화면에서는 본문을 읽은 뒤, 퀴즈를 풀기 전에 보는 위치가 기본입니다.
`storyNotes`는 본문 앞에서 먼저 노출해 학생이 핵심 설명보다 일화부터 읽게 만들지 않습니다. 목차와 본문 설명 사이에 끼워 넣지 않고, 본문 흐름을 보강하는 별도 영역으로 둡니다.
본문 소단원과 직접 연결되는 이야기는 `placement.type: "inline"`과 `chapterTitle`, `sectionHeading`을 사용해 해당 소단원 설명 뒤에 작은 박스로 배치합니다. 핵심 설명보다 먼저 나오면 안 되며, 반드시 관련 설명 뒤에 나와야 합니다.
본문 흐름을 끊을 만큼 길거나 특정 소단원에 붙이기 어려운 보조 이야기는 `placement.type: "afterBody"`로 두거나 `placement`를 생략해 본문 뒤의 `본문과 함께 보는 이야기` 영역에 둡니다.
전설, 설화, 의견이 분분한 이야기는 본문 문장 속에 섞지 않고 `storyNotes` 박스로 분리합니다. `debated` 또는 `attributed` 성격이 드러나야 하며, 실제 사실과 전승을 구분하는 문장을 포함합니다. 화면에서는 `전승으로 전해짐`, `실제 사실과 구분 필요` 같은 배지로도 구분합니다.
`storyNotes` 제목과 본문은 해당 지식의 기존 목차 중 어느 내용과 연결되는지 분명해야 합니다. 연결되는 본문 단원이 없다면 먼저 본문 구조가 부족한지, 아니면 storyNotes를 넣지 않는 편이 나은지 판단합니다.
`storyNotes` 제목은 본문 장 제목이나 소제목을 그대로 반복하지 않습니다. 예를 들어 본문 소제목이 `소금 행진`이면 이야기 제목은 `1930년 소금 행진`처럼 기록의 시기나 자료 성격이 드러나게 씁니다.
`storyNotes`가 본문과 같은 문장을 반복하거나 정의를 다시 풀어 쓰는 데 그치면 삭제하거나 더 구체적인 기록, 말, 전승으로 바꿉니다.
`storyNotes`에는 `title`, `type`, `reliability`, `body`, `sourceNote`를 둡니다. `type`은 `record`, `quote`, `anecdote`, `debated` 중 하나를 사용하고, `reliability`는 `confirmed`, `attributed`, `debated` 중 하나를 사용합니다.
`confirmed`는 공식 자료나 원자료로 확인되는 기록, `attributed`는 특정 인물에게 전해지지만 원문 확인 범위가 제한적인 말이나 이야기, `debated`는 실제 여부나 세부 내용에 의견이 분분한 이야기입니다.
의견이 분분한 이야기를 넣을 때는 흥미 위주로 단정하지 말고, 무엇이 확인되고 무엇이 전해지는지 함께 씁니다. 예: 뉴턴의 사과 이야기는 전해지는 기록이 있으나 “사과가 머리에 떨어졌다”는 식의 세부 장면은 널리 덧붙은 이야기로 구분합니다.
명언은 짧아도 출처를 확인합니다. 출처가 불확실한 명언은 넣지 않거나, `attributed`·`debated`로 표시하고 불확실성을 본문에 씁니다.
`쉽게 풀어보기`, `생활 속 예시`, `더 깊게 생각하기`, `다음 질문`처럼 어느 지식에나 붙일 수 있는 제목을 반복하지 않습니다.
`오해하기 쉬운 부분`, `뉴스에서 볼 때`, `생활 속 연결`처럼 너무 넓은 제목도 그대로 반복하지 말고, 해당 지식에서 무엇을 헷갈리기 쉬운지 드러나게 씁니다.
이미 뜻에서 충분히 쉽게 설명했다면 별도의 쉬운 풀이 단원을 만들지 말고, 예시·기준·오해·질문처럼 존재 이유가 분명한 단원으로 나눕니다.
목차는 지식마다 다를 수 있지만, 학생이 “왜 이 순서로 읽어야 하는지” 알 수 있어야 합니다.
각 작은 단원은 길이보다 역할을 기준으로 판단합니다. 개념, 기준, 예시, 주의점 중 필요한 내용을 충분히 설명했다면 짧아도 괜찮습니다.
한 지식 전체는 주제에 필요한 뜻, 원리나 배경, 예시, 오해 바로잡기 중 알맞은 요소를 갖추되, 필요 없는 내용을 억지로 늘리지 않습니다.
`퀴즈`는 객관식이어야 하며, 문제마다 질문, 보기 4개, 정답 번호, 해설을 넣습니다. 새로 작성하거나 크게 고친 퀴즈는 선택지별 해설인 `choiceExplanations`도 함께 둡니다.
퀴즈는 지식마다 직접 읽고 판단해서 만듭니다. 여러 지식에 같은 틀을 적용해 일괄 생성하거나, 단어만 바꿔 대량으로 넣지 않습니다.
질문은 무엇을 묻는지 분명해야 합니다. `뜻`, `영향`, `결과`, `활동`, `주의해서 기억할 점`처럼 답을 고르는 기준을 문제 문장 안에 직접 씁니다.
`다음 보기 중`, `본문 내용과 맞는 문장`, `특징이나 예시`처럼 생성된 문제처럼 보이는 표현을 반복하지 않습니다.
`설명으로 맞는 것은?`, `조심해야 할 점은?`처럼 범위가 넓어 보기를 읽기 전까지 무엇을 묻는지 알기 어려운 질문을 반복하지 않습니다.
좋은 예시는 `급식과 영양에 대해 알맞게 설명한 것은 무엇인가요?`, `6·25전쟁의 결과로 알맞은 것은 무엇인가요?`, `세종대왕의 업적으로 알맞은 것은 무엇인가요?`처럼 자연스러운 교실 문제 문장입니다.
오답 보기는 다른 지식의 정의를 가져와 채우지 말고, 해당 지식에 대한 그럴듯하지만 틀린 설명으로 씁니다.
예를 들어 범위를 너무 좁게 보거나, 원인과 결과를 뒤집거나, 중요한 조건과 예외를 빠뜨린 문장으로 구성합니다.
누가 봐도 관련 없는 보기나 모든 지식에 반복되는 범용 오답 문구는 사용하지 않습니다.
오답 보기임이 바로 드러나는 어색한 설명형 문장이나 `무조건`, `언제나`, `항상` 같은 단정 표현을 반복하지 않습니다.
해설은 “본문에서는 ~라고 설명합니다” 같은 범용 문구로 끝내지 말고, 왜 그 선택지가 맞는지와 어떤 기준을 함께 봐야 하는지까지 설명합니다.
해설도 평가성 표현을 피하고, 정답이 되는 사실 기준과 오답에서 놓친 조건을 설명합니다.
선택지별 해설은 보기 순서와 맞아야 합니다. 오답 해설은 선택한 오답의 잘못된 정보를 직접 지적해야 하며, 정답 해설을 그대로 복사하지 않습니다.
한 지식 안에서 여러 문제가 같은 정답 문장을 반복하지 않도록 합니다.
퀴즈 정답 문장은 현재 본문이나 정의에서 확인할 수 있어야 하며, 본문을 고치면 퀴즈도 함께 확인합니다.

빈칸 채우기 객관식은 `type: "blank"`를 사용하고, 질문에는 `____`를 넣습니다.
빈칸 문제는 지식마다 최대 1문항만 둡니다.
빈칸 문제는 핵심 용어, 연도, 인물, 원인과 결과처럼 본문을 읽은 뒤 확인할 가치가 있을 때만 추가합니다.
지식 내용이 짧거나 빈칸이 단순 암기만 만들면 3문항 또는 4문항을 맞추려고 억지로 추가하지 않습니다.
빈칸 문제의 정답과 오답은 해당 지식의 맥락에 맞게 직접 고릅니다. 보기만 바꿔 여러 파일에 자동으로 넣지 않습니다.
해설은 `빈칸에는 ~이 들어갑니다`처럼 자동 생성된 문장으로 끝내지 말고, 왜 그 말이 핵심인지 설명합니다.

## 6. 지도와 시각 자료

역사 지식에 지도가 필요할 때는 먼저 가져와도 되는 실제 지도 이미지나 지도 데이터를 찾습니다. 실제 지도 이미지를 그대로 사용할 수 있고 지식 내용과 맞는 경우에는 직접 다시 그리지 않습니다. 직접 제작이 필요한 경우에만 Natural Earth처럼 Public Domain 또는 공공데이터처럼 이용조건이 명확한 바탕지도 데이터를 사용하고, 공식 역사 자료로 지명, 위치, 이동 흐름, 시대 구분을 확인합니다.

- `figures` 배열에 지도, 도표, 그림 자료를 넣습니다.
- `asset`은 `assets/` 아래의 로컬 파일만 사용합니다. 외부 이미지 핫링크는 쓰지 않습니다.
- `dataSource`에는 지도 파일 또는 지도 데이터 이름, URL, 이용조건을 적어 지도 카드 안에서 출처가 보이게 합니다.
- `caption`에는 실제 국경선이나 이동 경로를 정밀하게 나타낸 지도가 아니라는 점을 필요하면 짧게 적습니다.
- 직접 제작한 지도라면 `sourceNote`에 바탕지도 데이터 출처, 역사 위치 확인 출처의 성격, 원문 지도나 사료 지도를 복제하지 않았다는 점을 반드시 적습니다.
- 외부 지도 이미지를 그대로 사용하는 경우에는 `sourceNote`에 지도 출처, 파일명, 저작자, 라이선스, 원본 파일을 수정 없이 사용했다는 점을 적습니다.
- 외부 기관의 이미지, 지도, 삽화, 표, 그래프는 공공누리 등 이용조건이 명확해도 데이터 이용조건과 게시 가능 여부를 먼저 확인합니다. 그대로 사용할 수 있는 조건이 확인된 경우에만 원본 파일을 사용합니다.
- 세력권 경계, 국경선, 전선처럼 시기별로 변하거나 학술 해석이 필요한 선은 충분히 검증하지 못한 상태에서 그리지 않습니다. 초등학생용 지식에는 주요 도시, 사건 위치, 이동 방향, 활동 지역 중심 지도를 우선 사용합니다.
- 외부 지도를 반드시 써야 하는 경우에는 원저작권자, 이용조건, 변경 가능 여부, 상업적 이용 가능 여부를 별도로 확인하고 출처 표기 방식을 기록합니다.

예시:

```json
"figures": [
  {
    "kind": "map",
    "title": "삼국 시대 지도",
    "asset": "assets/history-maps/imported/three-kingdoms-of-korea-map.png",
    "alt": "고구려, 백제, 신라와 가야가 표시된 삼국 시대 지도",
    "caption": "고구려, 백제, 신라와 가야의 위치 관계를 보여 주는 지도입니다. 위키미디어 공용의 원본 파일을 그대로 사용했습니다.",
    "dataSource": {
      "title": "File:Three Kingdoms of Korea Map.png",
      "url": "https://commons.wikimedia.org/wiki/File:Three_Kingdoms_of_Korea_Map.png",
      "license": "CC BY-SA 3.0"
    },
    "sourceNote": "지도 출처: Wikimedia Commons, File:Three Kingdoms of Korea Map.png. 저작자: Chris 73 / Wikimedia Commons. 라이선스: CC BY-SA 3.0. 트루핑키스 위키가 새로 그리거나 수정하지 않고 원본 파일을 사용했습니다."
  }
]
```

## 7. 출처 표기 형식

`sources`에는 문자열을 넣지 않고 아래 형식의 객체만 넣습니다.

```json
{
  "publisher": "기관명",
  "title": "자료명",
  "url": "https://example.go.kr/page",
  "usedFor": "이 자료로 확인한 내용",
  "license": "공식 자료 확인 후 학생용으로 새로 작성, 원문/이미지/표/그래프 복제 없음",
  "checkedAt": "2026-05-14"
}
```

`copyrightNote`에는 지식이 원문을 복제하지 않았고 학생용 문장으로 새로 작성되었다는 설명을 넣습니다.

## 8. 새 지식 추가 체크리스트

- 공식 자료 또는 이용 조건이 분명한 자료를 확인했는가?
- 교육과정과 연결되는 지식은 `data/source/curriculum-map.json`의 교과, 내용 영역, 학년 연결, 학교 공부 도움 방향도 함께 확인했는가?
- 출처 URL, 확인한 내용, 이용 메모, 확인일을 모두 적었는가?
- 원문 문장, 이미지, 표, 그래프를 가져오지 않았는가?
- 학생이 이해하기 쉬운 설명과 예시가 있는가?
- 제목, 본문, 교과, 분류, 태그, 검색 보조어로 검색될 수 있는가?
- 함께 보면 좋은 지식이 실제 존재하는 지식 ID로 연결되는가?
- 정보상자 없이 큰 단원, 작은 단원, 퀴즈 구조로 정리되었는가?
- `node scripts/validate-wiki-content.mjs`를 통과했는가?
